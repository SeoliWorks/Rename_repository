GitHubで管理しているプロジェクトの名前を変更したいです。ルートディレクトリのファイル名も変更したいです。<br>
どのような手順を取ればいいですか？<br>
<br>

GitHub上のプロジェクト名（リポジトリ名）と、ローカル環境のルートディレクトリ名（フォルダ名）の両方を変更する場合、**「GitHub側の設定変更」**と**「ローカル側のGit設定の更新」**の2つの作業が必要です。

手順をステップごとに解説します。

---

### 手順1: GitHubでリポジトリ名を変更する

まず、リモート（GitHub上）の名前を変更します。

1. GitHubのリポジトリページを開きます。
2. 上部のタブから **[Settings]**（設定）をクリックします。
3. "General" ページの最上部にある **"Repository name"** に新しい名前を入力します。
4. **[Rename]** ボタンをクリックします。

> **注意:** リポジトリ名を変更すると、GitHubのURLが変わります（例: `github.com/user/old-name` → `github.com/user/new-name`）。GitHubは古いURLからのリダイレクト（転送）を自動で行ってくれますが、手順3で正式なURLに更新することを推奨します。

---

### 手順2: ローカルのディレクトリ名を変更する

次に、あなたのパソコン上のフォルダ名を変更します。

1. 現在、そのディレクトリでターミナルやエディタを開いている場合は、一度閉じます。
2. Finder (Mac) や エクスプローラー (Windows)、またはコマンドラインを使って、**フォルダ名を直接変更**してください。

Gitの管理情報はフォルダの中にある `.git` フォルダ（隠しフォルダ）に格納されているため、**外側のフォルダ名を変更してもGitの履歴や管理には影響ありません。**

---

### 手順3: ローカルのGitリモートURLを更新する

ここが最も重要なステップです。GitHubでリポジトリ名（URL）が変わったため、ローカルのGitに「新しいアップロード先」を教える必要があります。

1. ターミナル（またはコマンドプロンプト）を開き、**名前を変更した新しいディレクトリ**に移動します。
```bash
cd path/to/新しいディレクトリ名

```


2. 現在のリモートURLを確認します。
```bash
git remote -v

```


*※ ここで古いリポジトリ名のURLが表示されるはずです。*
3. リモートURLを新しいリポジトリ名に更新します。
```bash
git remote set-url origin https://github.com/ユーザー名/新しいリポジトリ名.git

```


*※ SSHを使用している場合は `git@github.com:ユーザー名/新しいリポジトリ名.git` としてください。*
4. 正しく変更されたか確認します。
```bash
git remote -v

```


*※ 新しいURLが表示されれば成功です。*

---

### 手順4: プロジェクト内の設定ファイルを更新する（必要な場合）

ディレクトリ名やリポジトリ名が変わったことで、コード内の記述も修正が必要な場合があります。以下を確認してください。

* **パッケージ管理ファイル:** `package.json` (Node.js), `go.mod` (Go), `pom.xml` (Java) などの `name` フィールド。
* **ドキュメント:** `README.md` 内のプロジェクト名やバッジのリンク。
* **CI/CD設定:** GitHub Actionsなどの設定ファイルで、リポジトリ名をハードコードしている箇所がないか。

---

これで変更作業は完了です。試しに `git pull` や `git push` を行って、正常に通信できるか確認してみてください。
